1. Какво е HTTP ? - HTTP е мрежов протокол за пренос на данни.

2. Какво е HTTP response (oт какво е изграден) ? - HTTP respone е обратната
връзка от сървър към клиент. Изграден е от данните, които клиентът е поискал
и съдържа съобщение, че данните са успешно получени или че има error и данните
не са пристигнали.

3. Какво е HTTP request (oт какво е изграден) ? - HTTP request са данните,
които клиентът иска да вземе от даден сървър. Изграден е от части на URL-a,
които показват кои данни иска да получи клиентът.

4. Кои са HTTP методите (тези за които се сещате) ? - Всеки от методите работи
със сървър и всички операции се извършват не него.

   GET - взимане на данни
   PUT - записване на данни
   DELETE - триене на данни
   UPDATE - ъпдейтване на всички данни
   PATCH - ъпдейтване на посочени от нас данни

5. Какво е статус код в HTTP и от какво значение ? - Статус код в HTTP e
съобщение в response-a от сървъра към нас. Статус кодовете показват резултата
от нашия request за това как е минал.

6. Кои са често използваните статус кодове ? 
 
   HTTP Status Code 200 - OK. ...
   HTTP Status Code 301 - Permanent Redirect. ...
   HTTP Status Code 302 - Temporary Redirect. ...
   HTTP Status Code 404 - Not Found. ...
   HTTP Status Code 410 - Gone. ...
   HTTP Status Code 500 - Internal Server Error. ...
   HTTP Status Code 503 - Service Unavailable.

7. Каква е разликата между http и https ? - В https данните, които въвеждаме
в сайта са криптирани и това гарантира, че няма да има изтичане на данни,
докато в http не са криптирани.

8. Знаем ли каква е разликата между http1.1 и http2 (защото имаме нужда от подобрение) ?
  
   -http2 e доста по-бърз от http1.1, защото има мултиплексиране. Това значи,
    че всяка наша интеракция със сървъра се осъществява чрез една TCP връзка, а
    при http1.1 това става с нова връзка за всяко нещо, което искаме да 
    изпратим или получим и това го прави по-бавен протокол.
